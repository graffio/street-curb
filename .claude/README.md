# How We Work

## Workflow

1. **Brainstorm** - Discuss what to build (no templates)
2. **Plan** - Generate `current-task.json` using `tasks/plan-feature.md`
3. **Implement** - Follow `current-task.json` exactly
4. **Record** - Archive outcomes using `tasks/record-completion.md`

## Files

| File               | Purpose                   | When to read          |
|--------------------|---------------------------|-----------------------|
| `CLAUDE.md`        | Communication norms       | Session start         |
| `conventions.md`   | Code style                | When writing code     |
| `tasks/*.md`       | How to do specific things | When doing that thing |
| `current-task.json`| Active task spec          | During implementation |

## Key Rule

Templates are steps to follow, not rules to remember. If a convention matters, it's a step in a template.
